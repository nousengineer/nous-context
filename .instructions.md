# ThinkCoffee Development Guidelines

## Code Style & Conventions

### NO EMOJIS
- **NEVER** use emojis in code (backend, frontend, comments, or documentation)
- Replace all emojis with:
  1. SVG components from icon libraries (lucide-react, react-icons, heroicons)
  2. Text descriptions for clarity
  3. Set icon props to `null` as placeholder during development

Example: Instead of "📊 Dashboard", use `<DashboardIcon />` or plain text "Dashboard".

## Architecture

### Monorepo Structure
- pnpm workspace monorepo with packages: `core`, `mcp-server`, `cli`, `vscode`
- TypeORM + SQLite (stored at `~/.thinkcoffee/data.sqlite`)
- `synchronize: true` in dev — delete DB file to reset schema

### Build & Dependencies
- `pnpm build` builds all packages
- `pnpm test` runs `scripts/test-integration.js` (27 tests)
- Core must build first (others reference it via `workspace:*`)
- MCP server uses standalone tsconfig with `module: Node16, moduleResolution: Node16` (required for SDK subpath exports)
- CLI/vscode extend `tsconfig.base.json` (commonjs)

### Package Details
- **@thinkcoffee/core**: entities, services, database, export (6 formats), validation (zod), crypto
- **@thinkcoffee/mcp-server**: MCP server with resources, context tools, file tools (read/write/edit/list/search/structure); bin: `thinkcoffee-mcp`; workspace sandboxed via `--workspace=<path>` arg or `THINKCOFFEE_WORKSPACE` env
- **@thinkcoffee/cli**: Commander.js CLI; bin: `think`
- **thinkcoffee-vscode**: VS Code extension with tree views, commands (addFileAsContext, addSelectionAsContext, addStructureAsContext, openContextFile, etc.)

## Known Gotchas

### TypeORM & Reflection
- `emitDecoratorMetadata`: union types like `string | null` reflect as `Object` → must specify explicit column type (e.g., `@Column('text', { nullable: true })`)
- Cascade delete: `onDelete: 'CASCADE'` goes on `@ManyToOne` (child side), not `@OneToMany`

### SQLite Limitations
- SQLite can't ALTER FK constraints — delete DB file and let TypeORM recreate
- Use `synchronize: true` in dev for automatic schema updates

### Package-Specific Notes
- MCP SDK `ResourceTemplate` needed for templated URIs; callback signature is `(uri, variables, extra)` not destructured
- chalk v5 is ESM-only — removed from CLI (not needed)
